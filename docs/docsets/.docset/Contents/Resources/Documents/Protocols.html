The following protocols are available globally.

This is the Protocol that Obesrvers of the Views Must Use

  • LGV_UICleantimeImageViewObserverProtocol

    This protocol allows other objects to register as observers, and receive messages from the image view.

    There is only one message that the image view will send:

    • renderingComplete(view inImageView: LGV_UICleantimeImageViewBase).

    The images are generated in a separate thread (sort of -it’s the main thread), and this is called when rendering is complete.

    Note that this class extends the RVS_GeneralObserverSubTrackerProtocol protocol. This means that the observer needs to be a class (not a struct), and should define a couple of instance properties:

    • uuid: UUID
    • subscriptions: [RVS_GeneralObservableProtocol]
